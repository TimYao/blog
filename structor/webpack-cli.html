# webpack-cli (文章整理待更新中......)

webpack-cli针对webpack为开发人员根据自行提供了自定义webpack配置信息，快速的搭建基本webpack配置信息

# 命令分类

  • 命令参数：

init：初始化一个webpack配置

migrate: 升级一个新版本配置

serve: 开启webpack dev server

loader: loader脚手架

plugin: plugin脚手架

info: 输出安装系统以及依赖信息

  • 非命令参数

设定webpack的参数，例如entry

# 流程理解

  • process.argv 命令参数分解，分解出commander与options arg参数

  • commander命令为(package: init/migrate/serve/loader/plugin/info)，则利用require.resolve监测项目目录或者全局目录是否已安装@webpack-cli/package (package: init/migrate/serve/loader/plugin/info)包，若安装了，则require('@webpack-cli/package')(...arg);否则通过node模块readline模块交互性完成指定模块安装，利用node模块spawn开启子进程完成安装包，并require('@webpack-cli/package')(...arg)

  • 非commander命令，利用yarg命令工具完成参数预设和解析命令

  • 解析后处理基本webpack.config.js读取，并与命令行处理参数进行合并，并在完成参数的其他需求设定

  • 合并完成后的参数options,通过require('webpack')(options),传入并调用run方法完成启动，回调中完成响应的信息
